# RegexMaxLength 插件模板化改造计划

## 1. 创建公共模板文件
- **`public/plugin-template.html`**: 插件页面的 HTML 模板，包含通用的头部、输入、控制、信息区域
- **`public/css/plugin.css`**: 插件页面的通用样式（从 RegexMaxLength/style.css 中提取公共部分）
- **`public/js/plugin.js`**: 插件页面的通用 JavaScript 逻辑（控制按钮、速度控制等）

## 2. 修改 RegexMaxLength 插件
- **更新 `plugins/RegexMaxLength/index.html`**: 简化为只包含算法特定的内容（canvas 和特定输入）
- **创建 `plugins/RegexMaxLength/config.json`**: 存储插件元数据（标题、描述、复杂度、计算规则等）
- **简化 `plugins/RegexMaxLength/style.css`**: 只保留算法特定的样式

## 3. 实现动态渲染
- 在 `plugin.js` 中实现根据 JSON 配置动态渲染模板的功能
- 确保与现有的 transition.js 和 transition.css 兼容

## 4. 保持兼容性
- 确保改造后的页面功能与原来完全一致
- 保持现有的页面切换动画效果
