Core-Flow: 基于微内核协议的算法逻辑转换引擎
​声明： 本项目是我对“低代码可视化”场景的一次架构实践。核心目标是实现 算法逻辑 (Logic) 与 渲染指令 (UI) 的 物理隔离。



​ 为什么要做这个？ (The Problem)
​在维护传统可视化项目时，我发现将 Canvas 绘图逻辑硬编码在算法循环里是灾难性的。一旦算法逻辑变更（如 DFS 换成 BFS），整个 UI 渲染层都要重构。
​本项目通过一套微内核协议，将算法执行降维为“状态流”，实现了真正的插件化。
​


架构设计决策 (Engineering Decisions)
​1. 微内核 (Microkernel) 与 事件总线
​内核不关心你是“迷宫寻路”还是“八皇后”，它只负责调度插件的 生命周期 (Lifecycle)。
​指令队列 (Instruction Queue)：算法执行产生的每一步状态被封装成标准指令，通过 Emitter 异步推送到渲染队列。
​性能削峰 (Throttling)：针对高频 DFS 遍历，我在内核层实现了渲染节流，防止 144Hz 以上的无效重绘。

​2. 插件契约 (Plugin Contract)
​为了保证 0 耦合，所有算法插件必须遵循标准接口
​3. 自动化插件生成 (Code Generation)
​我实现了一个简易的 DSL 转换工具。用户在 Web 页面配置可视化元素，引擎会自动编译成符合规范的插件代码包，支持直接下载并热插拔运行。
