# 可视化编辑器框架设计方案

## 1. 创建编辑器页面结构

### 1.1 编辑器主页面 (`public/editor.html`)
- 左侧：组件面板（可拖拽的组件列表）
- 中间：可视化编辑画布
- 右侧：属性面板（编辑组件属性、命名编号）
- 底部：操作按钮（预览、生成插件模板）

### 1.2 编辑器样式 (`public/css/editor.css`)
- 编辑器布局样式
- 组件面板样式
- 画布样式
- 属性面板样式

## 2. 组件系统架构

### 2.1 组件基类 (`public/js/components/BaseComponent.js`)
- 组件ID、名称、位置、大小
- 渲染方法
- 序列化/反序列化方法

### 2.2 柱状图组件 (`public/js/components/BarChartComponent.js`)
- 实现动画接口：
  - `delete()` - 删除动画（右边向左靠近）
  - `insert(index, value)` - 插入动画
  - `swap(i, j)` - 交换动画
  - `highlight(indices, color)` - 染色动画
- 支持用户命名编号

### 2.3 组件注册中心 (`public/js/components/ComponentRegistry.js`)
- 组件注册/注销
- 组件工厂方法
- 组件类型管理

## 3. 编辑器核心逻辑

### 3.1 编辑器主类 (`public/js/editor/VisualEditor.js`)
- 拖拽功能实现
- 组件选择/编辑
- 画布管理
- 历史记录（撤销/重做）

### 3.2 属性编辑器 (`public/js/editor/PropertyEditor.js`)
- 组件属性编辑面板
- 命名编号编辑
- 动画预览

### 3.3 模板生成器 (`public/js/editor/TemplateGenerator.js`)
- 根据编辑内容生成插件模板
- 生成 `algorithm.js` 模板（包含动画API调用示例）
- 生成 `visualization.js` 模板
- 生成 `manifest.json` 和 `index.html`

## 4. 服务端支持

### 4.1 更新 `server.js`
- 添加插件生成 API (`POST /api/plugins/generate`)
- 接收编辑器数据并生成插件文件

## 5. 文件结构

```
public/
├── editor.html          # 编辑器页面
├── css/
│   └── editor.css       # 编辑器样式
└── js/
    ├── components/
    │   ├── BaseComponent.js
    │   ├── BarChartComponent.js
    │   └── ComponentRegistry.js
    └── editor/
        ├── VisualEditor.js
        ├── PropertyEditor.js
        └── TemplateGenerator.js
```

## 6. 使用流程

1. 开发者打开编辑器页面
2. 从左侧拖拽柱状图组件到画布
3. 在属性面板中命名组件编号（如 `bars`）
4. 点击"生成模板"按钮
5. 系统在 `plugins/` 目录生成插件模板
6. 开发者只需在 `algorithm.js` 中编写算法逻辑，使用 `bars.highlight(0, 'red')` 等API调用动画
